# 智能工时 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
智能工时是一款帮助用户自动记录、统计和分析工作时间的Web应用。通过语音或键盘输入，利用AI自动识别工作时间、工作内容和分类，让用户专注于工作本身，而非繁琐的记录工作。

### 1.2 核心价值
- **零门槛录入**：支持语音输入，说一句话就能记录
- **智能分类**：AI自动学习工作模式，智能归类统计
- **可视化分析**：清晰了解时间分配，优化工作方式

### 1.3 目标用户
- 需要记录和分析工作时间的职场人士
- 项目管理者需要了解团队时间分配
- 自由职业者需要追踪不同客户/项目的时间投入

## 3. 功能详细说明

### 3.1 首页 - 工作录入

#### 3.1.1 输入方式

**语音输入**
- 点击录音按钮开始录音
- 再次点击停止录音
- 自动将语音转为文字
- 实时显示录音时长和波形

**键盘输入**
- 点击切换到键盘输入模式
- 手动输入工作描述
- 支持自然语言描述（如"从上午9点到11点在做清洁机器人项目"）

#### 3.1.2 AI智能分析

用户输入后，AI自动分析并提取：

| 提取字段 | 说明 | 示例 |
|---------|------|------|
| 开始时间 | 工作开始时间（24小时制） | 09:00 |
| 结束时间 | 工作结束时间 | 11:00 |
| 工作时长 | 自动计算 | 120分钟 |
| 工作摘要 | 工作内容简短描述 | 清洁机器人调试 |
| 工作项目 | 匹配已有项目或创建新项目 | 清洁机器人项目 |
| 工作方式 | SOLO/会议/调研 | SOLO |
| 置信度 | AI判断可信程度 | 85% |

**工作方式识别规则**：
- `SOLO`：独自工作、专注开发、写文档、整理资料等独立工作（默认）
- `会议`：开会、讨论、评审、汇报、沟通等需要多人参与的工作
- `调研`：外出、拜访客户、实地考察、市场调研等需要外出的工作

**关键词识别**：
- 会议关键词：开会、会议、讨论、评审、汇报、沟通、协商、研讨
- 调研关键词：外出、拜访、考察、调研、实地、客户、出差、走访

#### 3.1.3 新项目检测

当AI识别到工作不属于已有项目时：
1. 弹出对话框提示用户
2. 显示AI识别的新项目名称（可手动修改）
3. 用户确认后创建新项目
4. 自动关联本次记录

#### 3.1.4 预览与编辑

**预览卡片显示内容**：
- 时间范围
- 工作时长
- 工作摘要
- 工作项目标签
- 工作方式标签（彩色）
- AI置信度

**编辑功能**：
- 修改开始/结束时间
- 修改工作内容
- 修改工作项目
- 修改工作方式
- 确认保存或取消

#### 3.1.5 首次使用引导

首次打开应用时显示欢迎页：
1. 功能介绍（语音录入、智能分类、可视化统计）
2. 工作项目设置（用户输入自己主要的几个工作项目）
3. 可跳过，之后在设置中添加

### 3.2 工作台 - 统计分析

#### 3.2.1 数据概览

**总工作时长**：所有记录的总时长

**工作记录数**：已保存的记录总数

#### 3.2.2 工作项目分布饼图

- 展示各工作项目的时长占比
- 不同项目用不同颜色区分
- 鼠标悬停显示具体时长和百分比

#### 3.2.3 工作方式分布饼图

- 展示SOLO/会议/调研的时长占比
- 颜色定义：
  - SOLO：绿色 (#10b981)
  - 会议：蓝色 (#3b82f6)
  - 调研：橙色 (#f59e0b)

#### 3.2.4 耗时趋势图

- 按日统计工作时长
- 折线图展示时间变化趋势
- 可查看最近一段时间的工作强度

#### 3.2.5 聚类统计

当标注记录达到5条以上时：
- AI自动分析记录内容
- 将相似的工作归为一组
- 显示聚类名称、记录数、总时长

#### 3.2.6 项目明细

- 列表形式展示各项目统计
- 显示项目名称、时长、占比
- 进度条可视化占比

---

### 3.3 待标注 - 快速标注

#### 3.3.1 待标注记录

- 显示未分配工作项目的记录
- 首页最多显示3条完整链接
- 待标注页面显示全部

#### 3.3.2 快速标注

- 每条记录下方显示所有工作项目按钮
- 点击即可快速标注
- 已标注记录自动移出此列表

---

### 3.4 设置 - 配置管理

#### 3.4.1 工作项目管理

**页面结构**：二级页面
- 返回按钮
- 进行中/已完成 标签页切换

**进行中标签**：
- 显示所有未标记完成的工作项目
- 每项显示：项目名称、使用次数
- 操作按钮：编辑、标记完成、删除

**已完成标签**：
- 显示已标记完成的工作项目
- 项目名称显示删除线效果
- 操作按钮：编辑、恢复、删除

**编辑功能**：
- 点击编辑按钮进入编辑模式
- 可修改项目名称
- 保存时自动更新关联记录中的摘要

**完成标记**：
- 点击勾选图标标记为已完成
- 已完成项目不会在AI分析时推荐
- 已完成项目的历史记录保留

#### 3.4.2 AI服务配置

**使用硅基流动API**：
- 默认使用Qwen2.5-7B-Instruct模型
- 用户需自行申请API Key

**配置流程**：
1. 点击"配置API Key"
2. 输入硅基流动API Key
3. 保存后立即生效
4. 可随时修改或清除

**离线模式**：
- 未配置API Key时使用基础分析模式
- 基础模式通过关键词匹配进行分类
- 建议配置API Key以获得更准确的识别

#### 3.4.3 数据管理

**导出数据**：
- 导出所有记录为JSON文件
- 包含工作项目配置、记录、聚类数据

**导入数据**：
- 从JSON文件恢复数据
- 覆盖现有数据

**清空数据**：
- 删除所有记录和配置
- 需要二次确认
- 不可恢复

## 4. 数据模型

### 4.1 工作项目 (WorkCategory)

```typescript
interface WorkCategory {
  id: string;              // 唯一标识
  name: string;             // 项目名称
  description: string;      // 项目描述
  keywords: string[];       // 关键词（用于匹配）
  usageCount: number;       // 使用次数
  createdAt: string;        // 创建时间
  isCompleted: boolean;     // 是否已完成
}
```

### 4.2 工作记录 (WorkRecord)

```typescript
interface WorkRecord {
  id: string;              // 唯一标识
  startTime: string;      // 开始时间（ISO格式）
  endTime: string;        // 结束时间（ISO格式）
  duration: number;       // 时长（分钟）
  rawText: string;        // 原始输入文本
  summary: string;        // AI生成的工作摘要
  categoryId: string | null;  // 关联的工作项目ID
  clusterId: string | null;   // 关联的聚类ID
  workMode: 'SOLO' | '会议' | '调研';  // 工作方式
  confidence: number;      // AI置信度
  annotationStatus: 'pending' | 'confirmed' | 'rejected';  // 标注状态
  createdAt: string;       // 创建时间
  updatedAt: string;       // 更新时间
}
```

### 4.3 用户配置 (UserConfig)

```typescript
interface UserConfig {
  hasCompletedOnboarding: boolean;  // 是否完成首次引导
  categories: WorkCategory[];       // 工作项目列表
  firstUseDate: string;             // 首次使用日期
  totalRecords: number;             // 总记录数
  annotationProgress: {             // 标注进度
    total: number;
    annotated: number;
  };
}
```

### 4.4 聚类 (Cluster)

```typescript
interface Cluster {
  id: string;              // 唯一标识
  categoryId: string;      // 关联的工作项目ID
  name: string;            // 聚类名称
  recordIds: string[];     // 包含的记录ID列表
  totalDuration: number;   // 总时长
  createdAt: string;       // 创建时间
  updatedAt: string;       // 更新时间
}
```

## 6. 用户旅程

### 6.1 首次使用

```
打开应用
    ↓
显示欢迎页
    ↓
用户介绍自己的主要工作项目
    ↓
完成设置，进入首页
```

### 6.2 日常使用

```
打开应用（直接进入首页）
    ↓
选择语音或键盘输入
    ↓
描述工作（如："上午9点到11点开项目评审会"）
    ↓
AI自动分析
    ↓
显示预览卡片（包含时间、内容、项目、会议标签）
    ↓
确认保存 或 编辑修改
    ↓
记录保存成功
    ↓
在工作台查看统计
```

### 6.3 遇到新工作

```
输入工作描述
    ↓
AI识别为新工作类型
    ↓
弹出对话框，显示识别的新项目名称
    ↓
用户确认或修改名称
    ↓
创建新项目并保存记录
```
