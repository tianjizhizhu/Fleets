# 智能工时统计工具 - 产品需求文档

## 一、产品定位与核心价值

### 1.1 产品定位

一款通用型语音输入工作耗时统计工具，通过自然语言交互让AI自动理解每位用户的工作结构，实现"说话即记录、智能即分类"的轻量化工作追踪体验。

### 1.2 核心价值主张

- **零预设分类**：不预设任何行业或岗位模板，每个用户的工作分类由用户自己定义
- **像人一样理解**：AI具备理解人类自然语言描述工作内容的能力，能够像同事交流一样理解"我今天在做XX项目"
- **越用越智能**：系统随着用户数据的积累，自动学习用户的工作模式，提供更精准的分类建议

### 1.3 目标用户

适用于任何需要对工作进行时间追踪和统计的人群：
- 项目制工作者（同时跟进多个项目）
- 自由职业者（服务多个客户）
- 企业员工（参与多个职能的工作）
- 任何希望了解自己时间分配的人

## 二、核心功能说明

### 2.1 首次使用引导

首次打开应用时，系统通过对话引导用户完成基础配置：

**对话示例：**
> 系统：欢迎使用！为了更好地帮你统计工时，请告诉我你都做哪些方面的工作？
>
> 用户：我是做保洁服务的，主要做清洁机器人、传统保洁、还有无人机这三块
>
> 系统：明白了，你的工作分为三大类：1. 清洁机器人 2. 传统保洁 3. 无人机。我记录下来了。如果你还有其他工作类型，随时可以告诉我。

**配置结果：**
- 用户用自己的语言定义工作类别
- AI学习用户的表述方式，理解类别之间的关系
- 支持随时新增、修改工作类别描述

### 2.2 语音输入记录

用户通过语音描述工作内容，系统自动解析并生成标准化记录：

**交互流程：**
1. 用户点击录音按钮，开始语音输入
2. 支持自然语言描述，如：
   - "从上午9点到11点在做清洁机器人项目"
   - "10点到10点半开了个项目会议"
   - "下午修了两小时无人机"
3. 录音过程中实时转写并显示
4. 录音结束后，AI自动提取信息生成记录卡片

**提取的信息：**
- 时间区间（起止时间或持续时长）
- 工作内容描述
- 所属工作类别（AI自动识别）

### 2.3 AI自动分类机制

**学习阶段（前期）：**
- 用户需要对每条记录手动指定或确认分类
- AI观察用户的选择模式，学习"什么样的工作内容属于哪个类别"
- 系统显示当前学习进度（如：已标注XX条记录，预计还需XX条触发自动分类）

**自动阶段（后期）：**
- 当AI识别到足够清晰的模式时，自动为新记录分类
- 用户可一键确认或修改分类
- 用户的每次修改都是AI的学习素材

**持续学习：**
- 用户可以随时添加新的工作类别："我最近开始做XX了"
- AI会记住这个新类别，并将后续相关内容归入此处
- 支持对类别描述进行修正

### 2.4 智能聚类统计

**聚类的意义：**
不是按行业预设分类，而是根据用户自己的数据，发现用户实际工作的细分模式：

**例如：**
用户A的"清洁机器人"工作可能被聚类为：
- 设备调试（多次短时记录：调参数、检查传感器等）
- 场地勘察（多次中等时长记录）
- 客户演示（偶发性长时记录）

用户B的"传统保洁"工作可能被聚类为：
- 日常清洁（规律性重复记录）
- 设备维护（偶发性记录）
- 团队培训（周期性记录）

**聚类结果展示：**
- 每个聚类卡片显示：聚类名称（AI生成或用户命名）、包含记录数、总耗时、占比
- 支持展开查看聚类包含的具体记录
- 支持手动调整聚类成员
- 支持对聚类进行重命名或拆分成多个聚类

### 2.5 工作台统计

**基础统计视图：**
- 各工作类别的总耗时与占比（饼图/环形图）
- 工作项明细列表（按耗时降序）
- 耗时趋势折线图（日/周/月粒度）

**高级视图：**
- 工作类别对比分析
- 高频工作项TOP10
- 工作时间分布（上午vs下午，周几最高效）

## 三、产品使用流程

### 3.1 新用户首次使用

```
打开应用 → 欢迎引导 → 描述工作类别 → 开始使用
```

### 3.2 日常记录流程

```
点击录音 → 语音描述工作 → 系统生成记录 → 确认/修改分类 → 保存
```

### 3.3 查看统计流程

```
进入工作台 → 选择时间范围 → 查看分类统计 → 查看聚类明细 → 可下钻至单条记录
```

## 四、数据模型设计

### 4.1 核心数据实体

**工作记录（WorkRecord）：**
- id：记录唯一标识
- startTime：开始时间
- endTime：结束时间
- duration：持续时长（分钟）
- rawText：原始语音转写文本
- summary：工作内容摘要
- categoryId：所属类别ID
- clusterId：所属聚类ID
- confidence：AI分类置信度
- annotationStatus：标注状态（pending/confirmed/rejected）
- createdAt：创建时间
- updatedAt：更新时间

**工作类别（WorkCategory）：**
- id：类别唯一标识
- name：类别名称（用户自定义）
- description：类别描述
- keywords：AI提取的关键词（用于辅助分类）
- usageCount：被使用次数
- createdAt：创建时间

**聚类结果（Cluster）：**
- id：聚类唯一标识
- categoryId：所属类别ID
- name：聚类名称
- recordIds：包含的记录ID列表
- totalDuration：总耗时
- createdAt：生成时间
- updatedAt：更新时间

**用户配置（UserConfig）：**
- categories：用户的工作类别列表
- firstUseDate：首次使用日期
- totalRecords：总记录数
- lastSyncTime：最后同步时间

### 4.2 数据存储

- 使用浏览器 LocalStorage 存储所有数据
- 支持数据导出为加密JSON文件
- 支持导入恢复

## 五、页面结构

### 5.1 页面清单

1. **首页/录音页**：核心录音功能，生成记录卡片
2. **记录列表页**：历史记录浏览，支持筛选、搜索
3. **标注页**：批量分类标注，查看标注进度
4. **聚类页**：查看和管理AI聚类结果
5. **工作台页**：统计图表展示
6. **设置页**：管理工作类别，导出/导入数据

### 5.2 页面关系

```
首页 ←→ 记录列表
  ↓
记录详情 ←→ 标注页
  ↓
聚类页 ←→ 工作台页
  ↓
设置页
```

## 六、技术方案

### 6.1 技术栈

- **前端框架**：Vue 3 + TypeScript
- **UI组件库**：Element Plus
- **状态管理**：Pinia
- **语音转写**：云端语音识别API（需接入）
- **AI能力**：大语言模型API（需接入）
- **本地存储**：LocalStorage + IndexedDB

### 6.2 关键技术点

**语音转写：**
- Web Audio API + MediaRecorder
- 实时流式转写，边录边显示
- 支持普通话/方言自适应

**AI理解能力：**
- 调用大模型API进行意图识别和实体提取
- 支持上下文学习，理解用户个性化的表述方式
- 定期批量处理，提升响应速度

**聚类算法：**
- 基于语义向量的相似度计算
- 层次聚类或K-means聚类
- 结合用户反馈持续优化

## 七、交互设计原则

1. **极简操作**：核心录音流程不超过3步
2. **即时反馈**：每个操作都有明确的状态反馈
3. **渐进式智能**：AI能力逐步释放，用户可控
4. **容错设计**：分类错误可随时修正
5. **隐私优先**：所有数据本地存储，语音不上传

## 八、开发优先级

### 第一阶段（MVP）
- 基础录音和语音转写
- 工作类别手动配置
- 记录生成与确认
- 基础记录管理

### 第二阶段
- AI自动分类
- 标注进度提示
- 聚类结果展示
- 基础统计图表

### 第三阶段
- 智能聚类优化
- 高级统计分析
- 数据导入导出
- 多维度筛选

## 九、成功指标

- 单次记录平均操作时间 < 30秒
- 语音识别准确率 > 90%
- AI自动分类准确率 > 80%
- 用户周均使用次数 > 3次
